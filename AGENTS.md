# 프로젝트 에이전트 가이드 (AGENTS.md)

이 문서는 AI·에이전트가 이 저장소에서 코드를 작성·수정할 때 **반드시 따를 규칙**과 **하지 말아야 할 것**을 정리합니다. 사용자가 매번 같은 설명을 반복하지 않도록 프로젝트 전제를 여기에 둡니다.

---

## 기술 스택 (고정)

| 영역 | 선택 |
|------|------|
| 프레임워크 | **Next.js** — **App Router** (`app/` 디렉터리) |
| 언어 | **TypeScript** (`.ts`, `.tsx`) |
| 스타일 | **Tailwind CSS** |
| 데이터베이스·백엔드 | **Supabase** (PostgreSQL, Auth, Storage 등 프로젝트에서 쓰는 범위 내) |

---

## 반드시 지켜야 할 것 (DO)

### Next.js (App Router)

- 라우팅·레이아웃·페이지는 `app/` 아래에 둔다. Pages Router(`pages/`)로 새 기능을 추가하지 않는다.
- 서버 컴포넌트를 기본으로 하고, `use client`는 브라우저 API·상태·이벤트가 필요한 최소 범위에만 쓴다.
- 데이터 페칭·폼 처리·리다이렉트 등은 App Router 패턴(`loading.tsx`, `error.tsx`, Route Handlers `route.ts` 등)에 맞춘다.
- 환경 변수는 `NEXT_PUBLIC_` 접두사 규칙을 지키고, 서버 전용 비밀은 클라이언트 번들에 노출되지 않게 한다.

### TypeScript

- 새 파일은 TypeScript로 작성한다. `any` 남발을 피하고, 필요하면 명시적 타입·`unknown`·제네릭을 사용한다.
- Supabase 클라이언트·쿼리 결과에는 가능하면 생성된 타입(Database types) 또는 명시적 인터페이스를 맞춘다.

### Tailwind CSS

- 스타일은 Tailwind 유틸리티 클래스를 우선한다. 인라인 `style`·별도 CSS 모듈은 프로젝트에 이미 그 패턴이 있을 때만 맞춰 사용한다.
- 반복되는 UI는 컴포넌트로 추출하고, `className`은 읽기 쉽게 정리한다.

### Supabase

- DB 접근은 Supabase 클라이언트(서버/클라이언트 용도 구분)를 통해 한다. 연결 정보는 환경 변수로만 둔다.
- RLS(Row Level Security)·정책을 존중하는 쿼리 패턴을 쓰고, 민감한 작업은 서버(Route Handler, Server Action 등)에서 처리하는 것을 우선한다.
- 마이그레이션·스키마 변경이 필요하면 Supabase 방식(SQL 마이그레이션 등)에 맞춰 제안하고, 임의로 로컬만의 스키마와 충돌 나게 작성하지 않는다.

### 공통

- 기존 코드 스타일·폴더 구조·이름 규칙을 우선 따른다.
- 사용자가 요청한 범위 안에서만 수정하고, 필요한 경우에만 의존성을 추가한다.

---

## 하면 안 되는 것 (DON'T)

### Next.js

- **`pages/` 라우터로 새 페이지·API를 추가하지 않는다.** (App Router만 사용)
- App Router와 혼용 목적으로 같은 경로를 `pages/`와 `app/`에 중복 정의하지 않는다.
- 서버 전용 비밀키를 `NEXT_PUBLIC_` 변수나 클라이언트 컴포넌트에 넣지 않는다.

### TypeScript

- 타입 안전성을 깨는 우회(`any` 남용, 불필요한 `@ts-ignore`)를 기본 선택으로 쓰지 않는다.

### 스타일

- **전역 CSS로 Tailwind와 중복·충돌하는 임의의 스타일 시스템**(예: 다른 유틸리티 프레임워크, 프로젝트에 없는 CSS-in-JS)을 사용자 요청 없이 도입하지 않는다.

### Supabase

- **서비스 역할 키(Service Role)**를 클라이언트 번들·`NEXT_PUBLIC_`·공개 저장소에 넣지 않는다.
- RLS를 우회한다고 가정하고 클라이언트에서만 “관리자 권한” 동작을 구현하지 않는다.

### 공통

- 요청과 무관한 대규모 리팩터·파일 삭제·의존성 무차별 추가를 하지 않는다.
- 사용자가 명시하지 않은 브라우저·Node 버전을 바꾸는 설정 변경을 임의로 하지 않는다.

---

## 요약

- **DO:** `app/` · TypeScript · Tailwind · Supabase를 일관되게 사용하고, 서버/클라이언트 경계와 보안을 지킨다.
- **DON'T:** Pages Router, 비밀 노출, 타입 무력화, 스택 외 기술의 임의 도입, 범위 밖 변경을 피한다.

이 문서와 충돌하는 요청이 있으면, 사용자에게 한 번 확인한 뒤 진행한다.
