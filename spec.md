# 오늘의 메뉴 추천 — 서비스 명세 (spec.md)

## 1. 개요

매일 식사 메뉴 선택으로 눈치를 보는 직장인 팀들이
**빠르고 공평하게** 메뉴를 결정할 수 있도록 돕는 웹 서비스.

- 팀원 누구나 먹고 싶은 메뉴를 제안
- 제안된 메뉴에 1인 1표로 투표
- 최다 득표 메뉴를 자동 선정하여 팀 전체에 공유

---

## 2. 기술 스택

| 영역 | 선택 |
|------|------|
| 프레임워크 | Next.js (App Router) |
| 언어 | TypeScript |
| 스타일 | Tailwind CSS |
| DB · Auth · Realtime | Supabase |

---

## 3. 사용자 흐름 (User Flow)

```
팀원 입장 → 로그인(또는 닉네임 입력) → 오늘의 세션 확인
  ├─ [제안 단계] 메뉴 제안 입력 → 등록
  ├─ [제안 마감] 모든 팀원 제안 완료 → 투표 단계 자동 전환
  ├─ [투표 단계] 10분 타이머 시작 → 메뉴 목록에서 1표 투표
  ├─ [투표 마감] 타이머 종료 또는 전원 투표 완료
  ├─ [결과 발표] 최다 득표 메뉴 선정 → 전체 팀원에게 결과 공유
  └─ [이력 조회] 최근 결정된 메뉴 목록 확인
```

## 4. 핵심 기능 상세

### 4-1. 팀 · 세션 관리

| 항목 | 설명 |
|------|------|
| 팀 생성 | 팀 이름 입력 → 초대 코드(6자리) 자동 생성 |
| 팀 참가 | 초대 코드 입력으로 참가 |
| 오늘의 세션 | 팀 당 하루 1세션, 날짜 기준 자동 생성 |
| 인원 확인 | 세션에 참여 중인 팀원 목록·인원 수 실시간 표시 |

### 4-2. 메뉴 제안

| 항목 | 설명 |
|------|------|
| 제안 입력 | 텍스트 입력 (예: "김치찌개", "초밥") |
| 중복 방지 | 같은 메뉴가 이미 있으면 알림, 중복 등록 불가 |
| 제안 표시 | 등록된 메뉴 카드가 실시간으로 목록에 추가 |
| 제안 마감 조건 | 모든 팀원이 최소 1개 이상 제안 완료 시 자동 마감 |

### 4-3. 투표

| 항목 | 설명 |
|------|------|
| 투표 시작 | 제안 마감 직후 자동 전환, 10분 카운트다운 시작 |
| 타이머 | 화면 상단에 **10:00 → 00:00** 스톱워치(카운트다운) 표시 |
| 투표 방식 | 제안된 메뉴 목록에서 **1인 1표** 선택 |
| 투표 변경 | 마감 전까지 변경 가능 |
| 투표 현황 | 각 메뉴의 현재 득표 수를 실시간 표시 |
| 투표 마감 | 타이머 종료 **또는** 전원 투표 완료 시 자동 마감 |

### 4-4. 결과 발표

| 항목 | 설명 |
|------|------|
| 선정 기준 | 최다 득표 메뉴 자동 선정 |
| 동점 처리 | 동점 시 동점 메뉴 중 랜덤 선정 |
| 결과 화면 | 선정된 메뉴를 강조 표시 + 전체 투표 결과 요약 |
| 팀 공유 | 결과 확정 시 참여 팀원 전원에게 실시간 알림(화면 갱신) |

### 4-5. 메뉴 이력

| 항목 | 설명 |
|------|------|
| 이력 목록 | 최근 결정된 메뉴를 날짜·메뉴명·득표 수와 함께 리스트 표시 |
| 조회 범위 | 최근 30일 (페이지네이션 또는 무한 스크롤) |

## 5. 데이터 모델 (Supabase)

### teams
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid (PK) | 팀 ID |
| name | text | 팀 이름 |
| invite_code | text (unique) | 6자리 초대 코드 |
| created_at | timestamptz | 생성 시각 |

### members
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid (PK) | 멤버 ID |
| team_id | uuid (FK → teams) | 소속 팀 |
| nickname | text | 표시 이름 |
| user_id | uuid (nullable, FK → auth.users) | Supabase Auth 연동 시 |
| created_at | timestamptz | 가입 시각 |

### sessions
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid (PK) | 세션 ID |
| team_id | uuid (FK → teams) | 팀 |
| date | date (unique per team) | 날짜 |
| status | text | `proposing` · `voting` · `completed` |
| vote_started_at | timestamptz (nullable) | 투표 시작 시각 |
| decided_menu | text (nullable) | 최종 선정 메뉴 |
| created_at | timestamptz | 생성 시각 |

### proposals
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid (PK) | 제안 ID |
| session_id | uuid (FK → sessions) | 세션 |
| member_id | uuid (FK → members) | 제안자 |
| menu_name | text | 메뉴명 |
| created_at | timestamptz | 등록 시각 |
| **unique** | (session_id, menu_name) | 같은 세션 내 메뉴 중복 방지 |

### votes
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid (PK) | 투표 ID |
| session_id | uuid (FK → sessions) | 세션 |
| member_id | uuid (FK → members) | 투표자 |
| proposal_id | uuid (FK → proposals) | 선택한 메뉴 |
| created_at | timestamptz | 투표 시각 |
| **unique** | (session_id, member_id) | 1인 1표 제약 |

## 6. 페이지 구조 (App Router)

```
app/
├─ page.tsx                     # 랜딩 — 팀 생성 / 참가
├─ team/[teamId]/
│  ├─ page.tsx                  # 팀 대시보드 (오늘의 세션으로 이동)
│  ├─ session/
│  │  ├─ page.tsx               # 오늘의 세션 (제안 → 투표 → 결과 통합 뷰)
│  │  └─ [sessionId]/
│  │     └─ page.tsx            # 특정 날짜 세션 결과 (이력 상세)
│  └─ history/
│     └─ page.tsx               # 메뉴 결정 이력 목록
├─ layout.tsx                   # 공통 레이아웃
└─ globals.css                  # Tailwind 전역 스타일
```

## 7. 실시간 처리

- **Supabase Realtime** (Postgres Changes)을 사용하여:
  - 제안 등록 시 목록 실시간 갱신
  - 투표 시 득표 수 실시간 갱신
  - 세션 상태 변경(제안→투표→완료) 실시간 반영
- 10분 타이머는 **클라이언트 카운트다운** + **서버 `vote_started_at` 기준 검증** 병행

## 8. 주요 UI 요소

| 요소 | 설명 |
|------|------|
| 메뉴 제안 카드 | 메뉴명 + 제안자 닉네임, 실시간 추가 |
| 투표 버튼 | 각 메뉴 카드에 투표 버튼, 선택 시 하이라이트 |

| 결과 화면 | 1위 메뉴 크게 강조 + 전체 순위 바 차트 |
| 이력 리스트 | 날짜 · 메뉴명 · 득표 수 테이블 형태 |

---

## 9. 제약 사항 · 예외 처리

- 팀원이 1명일 경우 투표 없이 제안 메뉴 바로 확정
- 제안 0건인 상태에서 투표 전환 불가
- 투표 마감 후 추가 투표 서버에서 거부
- 동점 시 랜덤 선정, 결과 화면에 "동점 → 랜덤 선정" 표기
- 날짜가 바뀌면 이전 세션은 자동 마감(status → completed)

- Supabase Auth 기반 로그인 (소셜 로그인)
